### **대댓글 기능을 어떻게 설계?**

- 요즘에는 깊은 depth의 대댓글은 사용하지 않음
- **_Self join_**이 필요함
- 일종의 트리구조가 될 것

<br>

| id  | pid(상위댓글) | depth |
| :-: | :-----------: | :---: |
|  1  |       0       |   0   |
|  2  |       1       |   1   |
|  3  |       1       |   1   |
|  4  |       2       |   2   |
|  5  |       3       |   2   |

```sql
select r1.id, r2.id, r2.pid
  from Reply r1 inner join Reply r2 on r1.id = r2.pid
 order by r1.pid
```
